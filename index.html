一张图看懂“火星坐标系”

发表于 2018-08-19 | 更新于 2018-11-17 | 分类于 地理信息科学
本文字数： 765 | 阅读时长 ≈ 1 分钟

不同坐标系的坐标在地图上的位置是否偏移与使用的地图框架(高德、百度、OpenLayes、Leaflet等)无关，而与所使用的底图供应商有关。

火星坐标系，也叫国测局坐标系（GCJ02），是由中国国家测绘局制订的地理信息系统的坐标系统。国内出版的各种地图系统（包括电子形式），必须至少采用“GCJ02”对地理位置进行首次加密。

A high-performance, feature-packed library for all your mapping needs.

OpenLayers 示例

介绍

这不是一个关于OpenLayers的入门教程，这只是一个关于OpenLayers的大量示例集锦。更多的是从OpenLayers Example直接搬过来的，那为什么要直接搬呢？看过 ArcGIS API for JavaScript 的朋友就知道，ArcGIS API for JS的文档不管是API还是例子，对开发者都很友好，有分类，看起来如行云流水，而 OpenLayers 提供的API文档和例子，没有分类，就是一个个的列表，让人无从下手，不能快速的将他们串起来。便以此为目的，“搬运”为途径，为初学者提供提供一条捷径。（当然，我也是初学者，因此才有了OpenLayers 示例）
除官网之外，我在GitHub上还找到了一个非常好的教程：OpenLayers 3 Primer。不仅有例子，还有详细的解释。
另外，关于OpenLayers的教程很多，不过，多数是基于 2 和 3 ，而我接触的时候已经是 4.6.5 了，而现在呢，不过1个月的时间，已经是 5.0.2 了。由于我喜欢新的东西，所以示例集锦中会有2个版本： 4.6.5 和 5.0.2。
